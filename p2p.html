About P2P

End-to-End Encryption
All messages are encrypted using AES-256-GCM before leaving your device. Your shared secret key derives the encryption key via PBKDF2 with 100,000 iterations. Not even the signaling server can read your data.
True Peer-to-Peer
Once connected, data flows directly between browsers using WebRTC. No cloud servers store or relay your messages—just a brief handshake through a signaling server to establish the connection.
Real-Time Sync
Text is transmitted as you type with minimal latency. Perfect for secure note sharing, credential exchange, or private conversations.
Zero Installation
A single HTML file with no dependencies to install. Runs entirely in your browser using native Web Crypto APIs—no accounts, no tracking, no data collection.
Shared Secret Model
Both parties agree on a secret passphrase offline. This passphrase generates both the connection ID and encryption key—no key exchange over the network.
How it works: Both users enter the same secret key → The key derives a unique room ID and AES-256 encryption key → Host waits, Joiner connects → All text is encrypted locally before transmission.
